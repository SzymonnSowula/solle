# Volle — Session 2026-05-17

## Co zrobiono

### 1. Obsidian vault structure
- Stworzono `/mnt/c/Users/szymo/Documents/Obsidian Vault/Volle/`
- Struktura: `00-Inbox`, `01-Agent-Lanes` (Backend/Frontend/Voice), `02-Architecture`, `03-Decisions`, `04-Sessions`, `05-TODO`, `99-Archive`
- ADR-001: raw asyncpg zamiast SQLAlchemy
- ADR-002: onboarding przed pierwszym użyciem

### 2. Backend — Postgres + DB layer
- Nowy modul `db/`: connection pool (asyncpg), migrations (raw SQL), repository (UserRepo, SettingsRepo, IntegrationRepo, SessionRepo)
- Tabele: users, settings, integrations, sessions, messages
- Automatyczne migracje przy starcie backendu
- Brak hardcoded wartosci — wszystko z DB

### 3. Onboarding API
- `GET /api/onboarding/status` — sprawdza czy onboarding zakonczony
- `POST /api/onboarding/profile` — zapisuje profil (name, company, tone, metrics)
- `POST /api/onboarding/integrations` — zapisuje integracje (Shopify, Allegro, Google Sheets, Baselinker)
- `POST /api/onboarding/complete` — finalizuje onboarding
- `GET/PATCH /api/settings/profile` — zarzadzanie profilem
- `GET/POST/DELETE /api/settings/integrations` — CRUD integracji

### 4. Persistent memory + conversation history
- Orchestrator laduje profil z DB zamiast `_MEMORY_PROFILE` hardcoded
- `run_pipeline` tworzy session i zapisuje user + assistant messages do tabeli `messages`
- WebSocket przekazuje `session_id` i zwraca go w response — mozliwosc kontynuacji rozmowy

### 5. Frontend — Onboarding + Settings
- Nowy komponent `Onboarding.svelte` — 5 krokow: welcome → profile → integrations → test → done
- Komponent `Settings.svelte` — edycja profilu i zarzadzanie integracjami
- `App.svelte` sprawdza onboarding status przy starcie, pokazuje onboarding lub widget
- Przycisk ⚙️ w widget otwiera settings
- API client w `lib/api.ts`

### 6. Realistyczne mock dane
- Nowy generator `generate_realistic_sales.py` — 90 dni, 1197 zamowien
- Sezonowosc: weekendowe spike (+60%), poniedzialkowy dip, wiosenny trend (marzec +25%), majowka (+35%)
- Realistyczne produkty z rozna sklonnoscia do zwrotow
- Statusy: completed/refunded/cancelled

### 7. Testy
- `tests/test_db.py` — testy repozytoriow (user, settings, integrations, sessions)
- `tests/test_onboarding_api.py` — testy endpointow onboarding/settings
- `tests/conftest.py` — konfiguracja pytest

### 8. Dokumentacja
- Zaktualizowano `HOWTORUN.md` — onboarding flow, testy, API examples
- Zaktualizowano `README.md` — project structure, onboarding, features, roadmap
