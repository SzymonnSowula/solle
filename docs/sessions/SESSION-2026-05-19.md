### 2. Phase D — Local LLM Offline Mode
Użytkownik może teraz przełączyć się na **lokalny model** (Ollama, LM Studio, llama.cpp) zamiast cloud API.

**Backend:**
- `agent/llm.py` zrefaktoryzowany — obsługuje dwa providery: `openai` i `local` przez ten sam kod `_chat_complete()`
- `LLM_PROVIDER=local` w `.env` lub per-request przez API
- Endpointy:
  - `GET /api/settings/llm/provider` — aktualny provider, model, URL
  - `POST /api/settings/llm/provider` — przełącz provider + zapisz w DB
  - `GET /api/settings/llm/local-status` — probe serwera lokalnego, lista modeli
- Przy starcie backend ładuje ostatni wybrany provider z tabeli `settings`
- Timeout dla local LLM: 120s (vs 30s cloud)
- `stream=false` w payload dla kompatybilności z lokalnymi serwerami

**Frontend:**
- `Settings.svelte` — nowa sekcja "LLM / Model AI"
- Toggle: "Użyj lokalnego LLM (Ollama/lm-studio)"
- Pola: URL serwera, nazwa modelu
- Przycisk "Sprawdź dostępność" — probe endpoint `/models`
- Wynik: liczba dostępnych modeli lub komunikat błędu

**Env vars (dodano do `.env.example`):**
```
LLM_PROVIDER=openai
LOCAL_LLM_BASE_URL=http://localhost:11434/v1
LOCAL_LLM_MODEL=llama3
```

## Jak przetestować Local LLM

1. Zainstaluj Ollama: https://ollama.com/download
2. Pobierz model: `ollama pull llama3`
3. Upewnij się że Ollama działa: `curl http://localhost:11434/v1/models`
4. W Settings Volle włącz "Użyj lokalnego LLM"
5. Kliknij "Sprawdź dostępność" — powinien pokazać liczbę modeli
6. Zapisz zmiany — od następnego zapytania Volle używa lokalnego modelu
