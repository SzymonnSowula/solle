# Volle — Session 2026-05-20

## Phase A — Rust Voice Runtime (zrobione)

### 1. Silero VAD (src/vad.rs)
- Zamieniono `webrtc-vad` na prawdziwy model ONNX (`silero_vad.onnx`) przez `tract-onnx`.
- API: `VadConfig`, `SileroVad::new(config)`, `process(chunk) -> (prob, speech_ended)`.
- `speech_ended` = true po ~300ms ciszy po wykrytej mowie.

### 2. Wake Word (src/wake_word.rs)
- ONNX path przez `tract-onnx` (jeśli model istnieje).
- Fallback na energy-gate stub gdy brak modelu.
- API: `WakeWordConfig`, `WAKE_WORD_WINDOW_SAMPLES` (2s @ 16kHz), `process(window) -> bool`.

### 3. Opus Encoder (src/encoder.rs)
- Feature gate `opus-encode` w Cargo.toml.
- Bez feature: PCM passthrough.
- Z feature: prawdziwy Opus via crate `opus` (i16 -> encode_vec_i16).
- API: `OpusConfig`, `OpusEncoder::new(config)`, `encode_f32(input) -> Vec<u8>`.

### 4. Pipeline (src/lib.rs)
- Zaktualizowano `VoicePipeline::run_blocking`:
  - Buforowanie okna wake-word (2s).
  - Buforowanie chunków VAD (512 samples).
  - WebSocket streaming z obsługą reconnect.
- Eksporty: `VadConfig`, `WakeWordConfig`, `OpusConfig`, `VAD_CHUNK_SAMPLES`, `WAKE_WORD_WINDOW_SAMPLES`.

### 5. Example (examples/capture_and_stream.rs)
- Naprawiono borrow-checker błędy (Arc<Mutex<Receiver>>).
- Dostosowano do nowych API (VadConfig by value, unwrap_or dla VAD result).

### 6. Misc
- Usunięto unused import `CloseFrame` w `ws_client.rs`.
- Usunięto śmieciowe pliki testowe (`test_tract.rs`).
